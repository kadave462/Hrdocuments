Net Salary Calculation — Documentation

A short, responsive explanation of how the calculator's forward and reverse logic was discovered and implemented.

How we discovered the calculation

The payroll calculation was reconstructed by inspecting payroll rules and iteratively validating them with sample payslips. Key observations:

  • LOGEMENT = 15% of SALBASE; TRANSPORT (cash) or TRANS NAT (in-kind) = 10% of SALBASE — exactly one is nonzero per employee. Cash transport enters BRUT; in-kind transport enters only the tax base (BTPR), not the employee's pocket.
  • BRUT includes SALBASE, ANC (seniority), LOGEMENT, TRANSPORT, HOUSE KEEPING, HS (overtime), PRIMES, PRIMEFONCTION, and REGURALISATION — but never TRANS NAT or ASS MAL.
  • Insurance has two independent dimensions: (1) split — G0–G2 default to 0%/100% (employer pays all), G3–G8 to 35%/65%; (2) premium base — 123,664 or 158,526 RWF, chosen per employee (not by grade). "Not insured" is always a valid option.
  • Employer-only costs (RSSB 2.3%, PENSION P 6%, ASS 60) are never deducted from an employee's net salary — they are tracked separately for the company's remittance obligations.
  • TPR is progressive on BTPR (0% → 10% → 20% → 30%). CBHI is 0.5% of BRUT after subtracting RSSB 0.3%, TPR, and PENSION E 6%. Both are employee deductions.
  • Reverse solving (NET SAL → SALBASE) uses binary search because TPR brackets make the formula non-invertible algebraically; the forward function is monotonic, so 60–80 iterations converge to within 0.01 RWF.

Formulas & reasoning

Below are the main formulas used by the calculator. These come directly from the rules observed in the spec.

Allowances and bases (computed in this order)

LOGEMENT  = SALBASE × 0.15
TRANSPORT = SALBASE × 0.10  (only if transport type = "cash"; else 0)
TRANS_NAT = SALBASE × 0.10  (only if transport type = "in-kind"; else 0)

BRUT = SALBASE + ANC + LOGEMENT + TRANSPORT
       + HOUSE_KEEPING + HS + PRIMES + PRIMEFONCTION + REGURALISATION
       -- TRANS_NAT and ASS_MAL are NOT included in BRUT

BCSR = BRUT − TRANSPORT          (social-security taxable base)
BTPR = BRUT + TRANS_NAT + ASS_MAL (tax / pension base)
       -- identity: BTPR = BCSR + 0.10×SALBASE + ASS_MAL

Contributions and taxes (employee deductions)

RSSB_employee  = BCSR × 0.003
TPR            = progressiveTax(BTPR)   -- see brackets below
PENSION_E      = BTPR × 0.06
CBHI           = (BRUT − RSSB_employee − TPR − PENSION_E) × 0.005
Employer-only costs — never deducted from NET SAL:
RSSB employer = BCSR × 0.023  |  PENSION P = BTPR × 0.06  |  ASS 60 (= ASS MAL)
Total employer cost = BTPR × 1.06 + BCSR × 0.023

Net salary

NET SAL = BRUT
        − RSSB_employee   (0.3% of BCSR)
        − TPR             (progressive, on BTPR)
        − CBHI            (0.5% of reduced base)
        − PENSION_E       (6% of BTPR)
        − ASS_40          (employee insurance share)
        − RESTAURANT      (450 RWF × canteen days)
        − PRET_VEHICULE   (car loan installment)
        − RBST_SFAR       (8% × (SALBASE+TRANSPORT+LOGEMENT) if loan active)
        − AUTRES_AVANCES

Reverse (solving NET SAL → SALBASE)

f(SALBASE) → NET_SAL is monotonically increasing.
Binary search over [0, 50,000,000]:
  60–80 iterations → precision < 0.01 RWF.

Function roots

insuranceCalc(split, base)

Returns [ASS_40, ASS_60] — the employee and employer insurance shares. Split defaults by grade: G0–G2 → "0/100" (employer pays all); G3–G8 → "35/65". Premium base is per-employee enrollment (123,664 = individual; 158,526 = family/dependents) — not grade-locked. "none" = not insured = [0, 0].

// split: '0/100' | '35/65' | 'none'
// base:  123664  | 158526
[ASS_40, ASS_60] = insuranceCalc(split, base)

// Results:
// '0/100' + 123664 → [0,       123664]
// '0/100' + 158526 → [0,       158526]
// '35/65' + 123664 → [43282,    80382]
// '35/65' + 158526 → [55484,   103042]
// 'none'           → [0,            0]

tprCalc(btpr)

Progressive income tax on BTPR (col X). Applied to both employee and employer pension base.

btpr ≤  60,000  →  TPR = 0
btpr ≤ 100,000  →  TPR = (btpr − 60,000) × 0.10
btpr ≤ 200,000  →  TPR = 4,000 + (btpr − 100,000) × 0.20
btpr >  200,000 →  TPR = 24,000 + (btpr − 200,000) × 0.30

calculate(salbase, opts)

Main payroll engine. Accepts SALBASE and optional inputs (ANC, transportType, insBase, split, loan flags, deductions). Returns an object with netSal, brut, btpr, bcsr, rssbE, tprVal, pensionE, cbhi, ass40, ass60, rbstSfar.

result = calculate(SALBASE, {
  transportType: 'cash' | 'in-kind',  // default 'cash'
  insBase: 123664 | 158526,
  split: '0/100' | '35/65' | 'none',
  loanActive: true | false,
  anc, houseKeeping, hs, primes,
  primefonction, reguralisation,
  restaurant, pretVehicule,
  rbstSfarManual, autresAvances
})

reverseCalc(targetNet, opts)

Binary search over SALBASE ∈ [0, 50,000,000]. Calls calculate() up to 80 times. Throws if target is below the minimum or above the maximum achievable net salary. Converges to within 0.01 RWF.

salbase = reverseCalc(targetNet, opts)
// same opts as calculate() — all deductions must be
// set correctly so the forward function matches reality
